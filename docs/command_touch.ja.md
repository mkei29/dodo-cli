# `touch`コマンド

`touch`コマンドは、Markdownファイルの管理を支援します。ファイルが存在しない場合は適切なフロントマターを含む新しいMarkdownファイルを作成し、既に存在する場合はフロントマターフィールドを更新します。これにより、ドキュメント全体で一貫したメタデータが保証されます。

## 使い方

```bash
dodo touch [flags]
```

## ユースケース
* 正しいフロントマター構造で新しいドキュメントファイルを素早く作成
* 手動編集なしで既存のファイルのタイムスタンプとメタデータを更新

## フラグ
* `-t, --title string`
  新しく作成されたファイルのタイトル。これはフロントマターの`title`フィールドを設定するために使用されます。

* `-p, --path string`
  ファイルのURLパス。これはフロントマターの`path`フィールドを設定し、ファイルがアクセス可能な場所を定義します。

* `--debug`
  デバッグモードを有効にします。トラブルシューティングのための追加出力を提供します。

* `--no-color`
  カラー出力を無効にします。カラーテキストをサポートしない環境で便利です。

* `--now string`
  RFC3339形式の現在時刻。これはフロントマターの`created_at`または`updated_at`フィールドを設定するために使用できます。

## フロントマター管理

`touch`コマンドを使用すると、Markdownファイルのフロントマターを管理でき、タイトル、パス、タイムスタンプなどのメタデータが一貫して適用されることを保証します。

## 例

```bash
# フロントマター付きの新しいMarkdownファイルを作成
$ dodo-cli touch example.md

# カスタムメタデータ付きのMarkdownファイルを作成
$ dodo-cli touch example.md --title "New Title" --path new-markdown
```
