ドキュメントのメンテナンスは開発フローにおいて非常に労力のかかる作業です。開発が進むにつれてドキュメントは古くなり、やがて誰にも読まれなくなる ── 多くの開発現場で起きていることです。

[dodo-doc](https://www.dodo-doc.com)は開発者向けの軽量で高機能なドキュメントホスティングサービスです。
アカウントを作成してCLIをインストールすれば、Markdownで書いたドキュメントをすぐに公開できます。

このページ自体もdodo-docでホストされています。他のページも見て、何ができるか確認してみてください。
以下のリンクからすぐに始められます。

https://document.do.dodo-doc.com/install

dodo-docは現在ベータ版で、無料プランのみ提供しています。

:::

# dodo-docができること

dodo-docは開発ワークフローに自然に組み込める設計で、ドキュメントを手軽に最新の状態に保てます。

## 閲覧者向け
* **見やすいモダンなUI**: 直感的に操作できるクリーンなデザインで、快適に読めます
* **ターミナルからアクセス**: コマンドひとつでドキュメントを開いたり検索できます。ターミナルから離れることなく作業できます
* **柔軟な公開設定**: ドキュメントを非公開にできます。個人メモや機密情報の扱いに最適です

## 作成者向け
* **開発者にやさしい**: Markdownを書いてアップロードするだけ。特別なフォーマットは不要です
* **簡単なセットアップ**: `dodo init`を実行するだけで設定ファイル（.dodo.yaml）を作成できます
* **CI/CD連携**: シングルバイナリのCLIをパイプラインに組み込んで、ドキュメントのデプロイを自動化できます

dodo-docはエディタとターミナルを中心とした開発スタイルに最適です。よく使うCLIコマンドはこちらです：

```bash
# 設定の初期化: 対話的に.dodo.yamlを作成
dodo init

# .dodo.yamlで定義されたドキュメントをアップロード
dodo upload

# クイックレビュー用の期間限定プレビューを公開
dodo preview

# ドキュメント一覧を表示してブラウザで開く
dodo docs

# ドキュメントを検索して結果を開く
dodo search
```

新機能を積極的に開発中です。今後の予定は[ロードマップ](/roadmap)をご覧ください。


## 他の選択肢

### 他のドキュメントホスティングサービス
[GitBook](https://www.gitbook.com/)や[Mintlify](https://www.mintlify.com/)といった選択肢もあります。
エンタープライズ向けの信頼性や高度なAI機能が必要な場合は、これらが適しています。

### 静的ファイルホスティングサービス
[mdBook](https://rust-lang.github.io/mdBook/)や[Fumadocs](https://fumadocs.dev/)などでHTMLを生成し、Cloudflare PagesやGitHub Pagesなどの静的ホスティングにデプロイする方法もあります。
GitHub Pagesなら、組織内のユーザーだけが見られるプライベートなドキュメントも作れます。
