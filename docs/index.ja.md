ドキュメントを常に最新に保つのは大変ですよね。開発は次々進むのに、ドキュメントは古くなり、読む人も離れていく...。
dodo-docは、ドキュメントをコードと同期させることで、この問題を解決します。

開発者のための、軽量でパワフルなドキュメントホスティングサービスです。
アカウントを作ってCLIをインストールすれば、数分でMarkdownを公開できます。

# dodo-docができること

dodo-docは、開発者のためのドキュメントホスティングサービスです。
開発ワークフローに自然に組み込めるよう設計されていて、ドキュメントを最新に保つのが簡単です。

## 読む人にとって
* **美しく、モダンなUI**: ナビゲーションしやすく、読みやすいクリーンなデザイン
* **ターミナルから直接アクセス**: シンプルなコマンドでドキュメントを開いたり検索できます。ターミナルを離れる必要なし
* **プライバシー管理**: 柔軟な表示設定でドキュメントを非公開にできます。個人メモや機密情報に最適

## 書く人にとって
* **開発者フレンドリー**: Markdownファイルを書いてアップロードするだけ。特別なフォーマットは不要
* **簡単セットアップ**: `dodo init`を実行するだけで、数秒で.dodo.yaml設定ファイルを作成
* **CI/CD統合**: 単一バイナリのCLIでパイプラインからドキュメントをデプロイ。自動化された信頼性の高いデプロイが可能

dodo-docは、エディタとターミナルで作業する人のために作られています。よく使うCLIコマンドはこちら：

```bash
# 設定の初期化: 対話的に.dodo.yamlを作成
dodo init

# .dodo.yamlで定義されたドキュメントをアップロード
dodo upload

# クイックレビュー用の期間限定プレビューを公開
dodo preview

# ドキュメント一覧を表示してブラウザで開く
dodo docs

# ドキュメントを検索して結果を開く
dodo search
```

より良いdodo-docを目指して、新機能を積極的に追加しています。今後の予定は[ロードマップ](/roadmap)をチェックしてください。


## 他の選択肢

### 他のドキュメントホスティングサービス
[GitBook](https://www.gitbook.com/)や[Mintlify](https://www.mintlify.com/)などの他のドキュメントホスティングサービスもあります。
信頼性や最先端のAI技術を使いたい場合は、これらのサービスがおすすめです。

### 静的ファイルホスティングサービス
[mdBook](https://rust-lang.github.io/mdBook/)や[Fumadocs](https://fumadocs.dev/)などのツールでドキュメント用のHTMLファイルを生成して、静的ホスティングサービスにアップロードする方法もあります。
Cloudflare PagesやGitHub Pagesなど、いろいろなサービスがあります。
GitHub Pagesなら、組織内のユーザーだけが見られるプライベートなドキュメントも作れます。
